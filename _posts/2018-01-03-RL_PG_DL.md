### Policy Gradient Methods

Policy gradient $\text{(PG)}$ methods use gradient ascent for maximizing the expected return $\boldsymbol{J_{\theta}}$. In gradient ascent, we update the parameters along the gradient $\nabla_{\boldsymbol{\theta}} \boldsymbol{J_{\theta}}$ as it points the direction of steepest ascent in the expected return. 

The accumulated reward,

$$
\begin{eqnarray}
&\text{Accumulated Reward : }& \quad \boldsymbol{R(\tau)} = r_{T}(\boldsymbol{x_t}) + \sum_{t=0}^{T-1}\boldsymbol{r_t(x_t, \mu_t)} \quad \text{or} \quad \boldsymbol{R(\tau)} = \sum_{t=0}^{\infty} \gamma^t r(\boldsymbol{x_t, \mu_t}) \\

&\text{Expected Accumulated Reward : }& \quad \boldsymbol{J(\theta)} = \mathbb{E}[\boldsymbol{R(\tau)}\mid \pi (\boldsymbol{\theta \mid \mu, \omega})] = \boldsymbol{\int} \boldsymbol{R(\tau)}p_{\theta}(\boldsymbol{\tau}) d\boldsymbol{\tau} \\
\end{eqnarray}
$$


The policy gradient update,

$$
\begin{eqnarray}
&\text{Policy Gradient Update : }& \quad \boldsymbol{\theta_{k+1}} = \boldsymbol{\theta_{k}} + \alpha \nabla_{\boldsymbol{\theta}} \boldsymbol{J_{\theta}}  \quad \text{with} \quad \alpha \quad \text{as learning rate}\\

&\text{Policy Gradient : }& \quad \nabla_{\boldsymbol{\theta}} \boldsymbol{J_\theta} = \boldsymbol{\int_{\tau} \nabla_\theta p_\theta (\tau) R(\tau) d \tau}
\end{eqnarray}
$$

### Finite difference methods

The finite difference policy gradient is the simplest way of obtaining the policy gradient. It is typically used with the episode-based evaluation strategy. The finite difference method estimates the gradient by applying small perturbations $\delta \boldsymbol{\theta}^{[i]}$ to the parameter vaector $\boldsymbol{\theta_k}$. We can either perturb each parameter value separately or use a probability distribution with samll variance to create the perturbations. For each perturbation, we obtain the change of the return $\delta J^{[i]} = R(\boldsymbol{\theta_k} + \delta \boldsymbol{\theta}^{[i]}) - R(\boldsymbol{\theta_k})$. For finite difference methods, the perturbations $\delta \boldsymbol{\theta}^{[i]}$ implement the exploration strategy in parameter space.

### Episode-Based Policy Gradients

**1. Likelihood-ratio policy gradients**

Likelihood-ratio methods were among the first policy search methods introduced in the early 1990s by Williams, and include the ***REINFORCE*** algorithm. These methods make use of the so-called "likelihood ratio" trick that is given by,

$$
\begin{eqnarray}
\boldsymbol{\nabla p(\theta) = p(\theta) \nabla \log p(\theta)}
\end{eqnarray}
$$

**2. Gaussian Policy Update Derivation**

Now, let's consider a Guassian policy with diagonal covariance, $p(\boldsymbol{\theta}) = \pi(\boldsymbol{\theta \mid \omega}) = \mathcal{N}(\boldsymbol{\mu}, \text{diag}(\boldsymbol{\sigma}^2))$, where $\boldsymbol{\omega = [\mu, \sigma]}$. Compute the gradients of the logarithm of the policy with respect to the parameters, $[\boldsymbol{\mu, \sigma}]$ analytically,

for mean $\boldsymbol{\mu}$,

$$
\begin{eqnarray}
\nabla_\mu \log \pi(\theta |\mu, \sigma) &=& \nabla_\mu \log \frac{1}{\sigma \sqrt{2 \pi}} \exp(-\frac{(\theta-\mu)^2}{2 \sigma^2}) \\
&=& -\nabla_\mu \frac{(\theta-\mu)^2}{2\sigma^2} \\
&=& \frac{(\theta-\mu)}{\sigma^2}
\end{eqnarray}
$$

for covariance $\boldsymbol{\sigma}$,

$$
\begin{eqnarray}
\nabla_\sigma \log \pi(\theta |\mu, \sigma) &=& \nabla_\sigma \log \frac{1}{\sigma \sqrt{2 \pi}} \exp(-\frac{(\theta-\mu)^2}{2 \sigma^2}) \\
&=& -\nabla_\sigma (\frac{1}{2} \log(\sigma^2) + \frac{(\theta-\mu)^2}{2\sigma^2}) \\
&=& \frac{(\theta-\mu)^2 - \sigma^2}{\sigma^3}
\end{eqnarray}
$$

**3. Baseline**

Because the gradient estimate can have a high variance, and subtracting a baseline can reduce the variance, we can always subtract a baseline from the gradient. Meanwhile the estimation can still remain unbiased. 

$$
\begin{eqnarray}
\nabla_\boldsymbol{\omega} J(\boldsymbol{\theta}) &=& \nabla_\boldsymbol{\omega} \int \pi(\boldsymbol{\theta; \mu, \sigma}) (R_\boldsymbol{\theta} -b)d\boldsymbol{\theta}\\
&=&\int \nabla_\boldsymbol{\omega} \pi(\boldsymbol{\theta; \mu, \sigma}) (R_\boldsymbol{\theta} -b)d\boldsymbol{\theta} \\
&=&\int \pi(\boldsymbol{\theta; \mu, \sigma}) \nabla_\boldsymbol{\omega} \log \pi(\boldsymbol{\theta; \mu, \sigma}) (R_\boldsymbol{\theta} - b)d\boldsymbol{\theta} \\
 &\approx& \sum_{i=1}^{N} \nabla_\boldsymbol{\omega} \log \pi (\boldsymbol{\theta_i; \omega}) (R_i - b)
\end{eqnarray}
$$

A good baseline can be the average reward. However, the returns can still have a lot of variance, $R_\theta = \mathbb{E}[\sum_{t=1}^{T}r_t \mid \boldsymbol{\theta}]$, because it is the sum over T random variables.

**4. Results**

$$
\begin{eqnarray}
\Longrightarrow \quad \nabla_\boldsymbol{\mu}  J_\omega &\approx& \sum_{i=1}^{N} \frac{1}{\text{diag}(\boldsymbol{\sigma}^2)} (\boldsymbol{\theta_i - \mu})(R_i-b)\\

\Longrightarrow \quad \nabla_\boldsymbol{\sigma}  J_\omega &\approx& \sum_{i=1}^{N} ( \frac{1}{\text{diag}(\boldsymbol{\sigma}^3)} (\boldsymbol{\theta - \mu})^2 - \frac{1}{\boldsymbol{\sigma}}) (R_i-b)
\end{eqnarray}
$$

### 2-DoF plannar robot example 

For more details about this 2-Dof plannar robot environment, which was originally developed by <a target="_blank" href="http://www.ias.tu-darmstadt.de/">  Intelligent Autonomous System Group </a> at TU Darmstadt, please visit my another blog  <a target="_blank" href="https://yun-long.github.io/2017/12/RL_ML/"> Policy Search > Maximum Likelihood Estimation</a>.

{% highlight python %}
env = Pend2dBallThrowDMP()

num_dim = 10
num_samples = 25
max_iter = 100
num_trials = 10

alpha_coeffs = [0.1, 0.2, 0.4]
mean_reward = np.zeros(shape=(max_iter, num_trials, len(alpha_coeffs)))

for l, alpha_coeff in enumerate(alpha_coeffs):
    for k in range(num_trials):
        Mu_w = np.zeros(shape=num_dim)
        # THIS TIME Sigma_w IS THE STD, NOT THE COVARIANCE! WE LEARN A DIAGONAL COV MATRIX!
        Sigma_w = np.eye(num_dim) * 1e1
        for i in range(max_iter):
            # sample
            rewards = np.zeros(shape=num_samples)
            theta_samples = np.random.multivariate_normal(mean=Mu_w, cov=Sigma_w, size=num_samples)
            for j, theta_sample in enumerate(theta_samples):
                rewards[j] = env.getReward(theta=theta_sample)
            mean_reward[i, k, l] = np.mean(rewards)

            # Compute Gradient
            Std_w = np.diag(Sigma_w)
            diff = theta_samples - Mu_w
            d_log_pi_Mu = diff * (1. / Std_w*2)
            d_log_pi_Std = diff * 2 / Std_w**3 - 1. / Std_w
            d_log_pi_omega = np.hstack((d_log_pi_Mu, d_log_pi_Std))

            # subtract a baseline
            advantage = rewards - np.mean(rewards)
            G = np.dot(advantage, d_log_pi_omega) / num_samples

            # normalize variance gradient
            G_sigma = G[num_dim:]
            G_sigma = G_sigma / np.linalg.norm(G_sigma)
            G[num_dim:] = G_sigma

            # update mean and covariance
            Mu_w = Mu_w + alpha_coeff * G[:num_dim]
            Sigma_w = Sigma_w + alpha_coeff * np.diag(G[num_dim:])

            # terminal conditions
            if (i > 0 and abs(mean_reward[i, k, l]-mean_reward[i-1, k, l]) < 1e-3) or (i == max_iter-1):
                mean_reward[i:, k, l] = mean_reward[i, k, l] # fill missing iter
                break
