### Introduction 

**[Watch this 12 Minutes Intro video on Youtue](https://www.youtube.com/watch?v=YFl2mCHdv24&t=0s)**.

Docker is an open-source software that enables developers to build, ship, and run distributed projects, whether on laptops, data center VMs, or the cloud, or whether on Windows, Linux, or Mac OS. 

Docker packs your project into an **image**, and run the project in a loosely isolated environment called **container**.

<img src="/images/posts/Docker/archetecture.png" height="269" width="500">  

Docker uses a client-server architecture. 

> * Client \\
	* The *terminal.app* you used to interact with Docker. 
* Docker Host
	* Docker Daemon
		* It listens for Docker API requests and manages Docker objects such as images, containers.
	* Docker Images
		* An image is a read-only template with instructions for creating a Docker container. 
	* Docker Containers
		* A container is a runnable instance of an image. 
	* Docker Services
		* Services allow you to scale containers across multiple Docker daemons. 
* Docker Registry 
	* It stores official images, e.g., ubuntu, python image
	* Or it stores personal public images, e.g., your hello_world image. 


### Installation for Mac
Docker installation on Mac is painless. 

Firstly, download the Docker Community Edition for Mac from **[docker store](https://store.docker.com/editions/community/docker-ce-desktop-mac)**. There you have two options, 

> * Stable - it is fully baked and tested. 
* Edge - it comes with latest features and experimental features turned on. 

Secondly, double-click *Docker.dmg* to start the install process. There you go, you have Docker on you Mac now.\\
Thirdly, test your installation by opening the *terminal.app* and typing following commands:

> * $ docker --version
* $ docker info

### Docker Python example 

Now, you have *Docker* installed on your Mac, and you want to run a python script *hello.py*. 

***Step 01***: create a *hello.py* script and type the following code in it.

{% highlight python %}
def hello():
	print("hello docker!")

hello()
{% endhighlight %}

***Step 02***: Open **[docker hub](https://hub.docker.com/)**, search the *python image* we want to use, which has official version. 
<img src="/images/posts/Docker/python.png" height="200" width="1000">  

***Step 03***: Pull the ***python image*** to your local computer, by typing the command on your terminal. The docker will start to download the *image*. \\
<img src="/images/posts/Docker/pythondown.png" height="150" width="1000">  
After the download process is finished, you can check the images in your docker by:
<img src="/images/posts/Docker/pythondone.png" height="50" width="1000"> 

> * $ docker pull python
* $ docker images
* $ docker images -a

***Step 04***: Eventually, you can run your *hello.py* by typing the following command:

> * docker run -v /Users/yunlong/Github/yun-long.github.io:/app python python3 /app/hello.py.
	* *run* create a container for *python* image. 
	* *-v* opition will mount the volume in the directory of your host machie, e.g.: /Users/yunlong/Github/yun-long.github.io, to a directory of the container you just create.
	* *python* is the image you download.
	* *python3 /app/hello.py* is to run the script in the container.

<img src="/images/posts/Docker/hello.png" height="40" width="1100"> 

***Congratulations! You have ran a python script using docker.***
