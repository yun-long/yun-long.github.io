### Introduction 

**[Watch this 12 Minutes Intro video on Youtue](https://www.youtube.com/watch?v=YFl2mCHdv24&t=0s)**.

Docker is an open-source software that enables developers to build, ship, and run distributed projects, whether on laptops, data center VMs, or the cloud, or whether on Windows, Linux, or Mac OS. 

Docker packes your project into an **image**, and run the project in a loosely isolated environment called **container**.

<img src="/images/posts/Docker/archetecture.png" height="269" width="500">  

Docker uses a client-server architecture. 

> * Client \\
	* The *terminal.app* you used to interact with Docker. 
* Docker Host
	* Docker Daemon
		* It listens for Docker API requests and manages Docker objects such as images, containers.
	* Docker Images
		* An image is a read-only template with instructions for creating a Docker container. 
	* Docker Containers
		* A container is a runnable instance of an image. 
	* Docker Services
		* Services allow you to scale containers across multiple Docker daemons. 
* Docker Registry 
	* It stores official images, e.g., ubuntu, python image
	* Or it stores personal public images, e.g., your hello_world image.
