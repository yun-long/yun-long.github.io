**Artificial Intelligence** has been achieving great successes in recent years, e.g., natural language processing or understanding, game playing, computer vision, ect. Among all the state-of-the-art approaches, __reinforcement learning__ is a type of unsupervised machine learning, which allows agents to automatically determine the optical policy based on feedback from the enviornment, in order to maximize the cumulative reward.

### Reinforcement Learning 
This is the description of a basic framing of a reinforcement learning scenario from wikipedia, in which a agent explores the enviornment throung a set of actions or policies and retrives the representations of a interpreted reward and state. The agent has to decide the optimal policy (actions) to collect as much reward as possible.  
<img src="/images/posts/RL_intro/agent.png" height="300" width="290">  
__Sequential Decision Making__ : The ultimate goal is to select actions to maximise total future reward. A reinforcement learning agent interacts with its environment in discrete time steps. At each time \\(t\\), the agent receives an observation \\( o\_t\\), which typically includes the reward \\(r\_t\\). It then chooses an action \\(a\_t\\) from the set of actions available, which is subsequently sent to the environment. The enviornment moves to a new state \\(s\_{t+1}\\) and the reward \\(r\_{t+1}\\) associated with the transition \\( (s\_t, a\_t, s\_{t+1})\\) is determined. However, actions may have long term consequences and the reward may be delayed. It may be better to sacrifice immediate reward to gain more long-term reward,  e.g., a financial investment may take months to mature.  

__Enviornment__ : The enviornment can be fully observable or partially observable. In fully observable enviornment senario, we assume the agent knows all the states and the interprated reward for a given action. Whereas in the opposing case, the agent can only observe its surrounding enviornment.

__Major Components of an Reinforcement Learning Agent__: 

> * Policy: agent's behaviour function, e.g., stochastic policy: \\( \pi(a|s) = \mathbb{P}[A\_t = a | S\_t = s] \\)
> * Value Function: how good is each state and / or action, e.g., \\(v\_{\pi}(s) = \mathbb{E}\_{\pi} [R\_{t+1} + \gamma R\_{t+2} + \gamma^2 R\_{t+3} + ... | S\_t = s ] \\)
> * Model: agent's representation of the enviornment, it predicts what the enviornment will do next, e.g., $$ P\_{ss'}^a = \mathbb{P} [S\_{t+1}=s'|S\_t = s, A\_t=a]$$


### Big Picture
<img src="/images/posts/RL_intro/bigpicture.png" height="350" width="340">  
The RL agent can be categorized in two different ways. One way is to classify the agent based on the value funtion and policy.

> * Value Based
	* No policy
	* Value Function
* Policy Based
	* Policy
	* No Value Function
* Actor Critic
	* Policy
	* Value Function

Another way is to classify it based on the model. 

> * Model Free 
	* Policy and /or Value Function
	* No Model
* Model Based
	* Policy and /or Value Function
	* Model

<img src="/images/posts/RL_intro/classes.png" height="350" width="400">  

Exploration vs. Exploitation (it is important to find a trad-off between these two strategies):
> * Exploration: finds more information about the enviornment
* Exploitation: exploits known information to maximise reward.
